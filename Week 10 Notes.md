# L9.2 - Version Control p2
All the git commands learn from somewhere else
Generated by ChatGPT
## 1. Setup (One-Time)

### Configure your identity

`git config --global user.name "Your Name" git config --global user.email "you@example.com"`
## 2. Starting a Project

### Initialize Git in a folder

`git init`

Creates a `.git` directory and starts tracking versions.

### Clone an existing repo

`git clone <repo-url>`

## 3. Checking Status / History

### Check what changed

`git status`

### See file differences

`git diff`

### See commit history

`git log`

Short, one-line log:

`git log --oneline`

---

## 4. Adding & Committing

### Add files to staging area

`git add <filename> git add .          # add all changes`

### Commit changes

`git commit -m "message"`

### Add & commit in a single shot

`git commit -am "message"`

(Only works for already-tracked files.)

## 5. Branching

### Create a new branch

`git branch <branch-name>`

### Switch to a branch

`git checkout <branch-name>`

### Create + switch

`git checkout -b <branch-name>`

### List branches

`git branch`
## 6. Merging

### Merge another branch into current

`git merge <branch-name>`

## 7. Push / Pull (Remote Sync)

### Check remote URLs

`git remote -v`

### Add a remote

`git remote add origin <url>`

### Push branch to remote

`git push origin <branch>`

### Pull latest changes

`git pull`

(_pull = fetch + merge_)

### Fetch without merging

`git fetch`

## 8. Undo / Fix

### Unstage a file

`git reset <file>`

### Undo local changes to a file

`git checkout -- <file>`

### Go back to a previous commit (soft

`git reset --soft <commit-id>`

### Go back (hard delete changes

`git reset --hard <commit-id>`

⚠️ _Dangerous: deletes work._

## 9. Stashing

Temporarily save changes without committing.

### Stash your work

`git stash`

### Bring back stashed changes

`git stash pop`

## 10. Useful Shortcuts

### Show commit + diff in one

`git show`

### Remove a tracked file

`git rm <file>`

### Move/rename a file

`git mv old new`

# L10.5 - Managing Storage
## LVM
* Logical Volume Managemnet
* pooling multiple storage devices as a single logical volume
* lvm2 tools: create and manage virtual block devices from physical devices

## RAID
* Distributing data over multiple discs

Modes
* RAID 0 - striping 
	* 2 min
	* speed up
* RAID 1 - Mirroring
	* 2 min
	* read is n times faster
	* n-1 drive failures tolerated
* RAID 5 - Block level parity with distributed parity
	* min 3
	* 1 drive failure tolerated
	* read is n times faster
	* write is n-1 times faster
* RAID 6 - Block level striping with dual distributed parity
	* main 4
	* 2 fails tolerated
	* read n times faster
	* write n-2 times faster
